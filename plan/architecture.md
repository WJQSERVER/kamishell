# 系统架构设计提案

## 1. 核心组件
*   **Lexer (词法分析器)**: 负责将输入的字符串流分解为 Token。需要识别 Shell 特有的符号（|、>、&）和编程语言特有的字面量。
*   **Parser (语法分析器)**: 采用递归下降算法构建抽象语法树 (AST)。能够处理命令调用、管道流、控制结构（if/for）等。
*   **AST (抽象语法树)**:
    *   `CommandNode`: 代表一个可执行命令。
    *   `PipeNode`: 代表管道连接。
    *   `ControlNode`: 代表 if/for 等逻辑控制块。
*   **Interpreter/Runtime (执行引擎)**:
    *   负责遍历 AST 并执行任务。
    *   管理变量作用域 (Scopes) 和环境变量。
    *   处理内置命令 (Built-ins) 与外部进程调用。

## 2. 跨平台支持
*   **路径处理**: 内部统一使用 Go 的 `path/filepath` 处理路径，在不同操作系统间自动转换分隔符。
*   **进程执行**: 使用 Go 标准库的 `os/exec`，并针对 Windows 和 Unix-like 系统处理不同的信号量和进程属性。
*   **内置命令重写**: 为了保证一致性，常用的基础工具（如 `ls`, `pwd`, `print`）将作为 Shell 内部的 Go 函数实现，而不是依赖系统自带的二进制文件。

## 3. 并发模型
*   利用 Go 的 Goroutines 实现非阻塞的后台任务。
*   支持传统的 Shell `&` 符号，同时也支持 `go` 关键字。
