# Kamishell 语言规范 (Spec)

## 1. 类型系统 (Type System)

Kamishell 是动态类型的，支持以下核心对象类型：

*   **INTEGER**: 基础 64 位有符号整数。
*   **STRING**: 字符串对象，支持 `+` 拼接。
*   **BOOLEAN**: `true` 或 `false`。
*   **FUNCTION**: 存储参数列表、函数体和词法环境。
*   **NULL**: 表示空值（关键字 `nil`）。
*   **ERROR**: 专门用于存储操作失败信息的结构化对象。

### 1.1 变量类型约束
Kamishell 可以为变量记录类型约束，`nil` / `NULL` 作为一种类型参与约束。

*   `var x int` 会为变量建立显式类型约束。
*   `x := 1` 会根据初始值推断类型并建立类型约束。
*   `x := nil` 会将变量类型约束为 `nil`；后续赋值必须也是 `nil`。
*   `var x`（无类型无值）是语法错误，必须提供类型或初始值。
*   `=` 会优先更新最近作用域中的同名变量，并遵守其已有类型约束。

### 1.2 Error 类型
运行时通过全局 `err` 变量自动暴露最近一次命令的结果。
*   **字段**:
    *   `Message`: 错误描述字符串。
    *   `Code`: 退出状态码 (exit code)。
    *   `Op`: 产生错误的命令或操作名称。
*   **布尔上下文**: 非空 Error 对象在判断中被视为 `true`。

## 2. 语法构造 (Grammar)

### 2.1 赋值与作用域
*   `:=`: 声明并赋值。如果在当前作用域不存在，则在当前作用域新建变量。
*   `=`: 重新赋值。如果变量不存在，则递归查找上级作用域直到全局。
*   支持词法作用域（Lexical Scoping）。

### 2.2 流程控制
*   **If-Else**: 强制使用大括号，大括号需跟在关键字同一行。
*   **For**: 支持带条件的循环。
*   **Func**: 函数定义支持位置参数。

### 2.3 命令组合
*   **管道 (|)**: 标准输出流连接。
*   **逻辑运算符 (&&, ||)**: 基于退出码的短路执行。
*   **异步执行 (&)**: 后缀符号，将整行转入后台协程。
*   **Go 关键字**: 前缀符号，将后续代码块或命令转入后台协程。

## 3. 运行时行为 (Runtime)

### 3.1 变量查找顺序
1.  本地作用域（函数内部）。
2.  外部闭包作用域。
3.  环境变量。
4.  Shell 内置命令库。
5.  系统 PATH。

### 3.2 作业控制 (Job Control)
*   所有后台任务在 `builtin.Jobs` 全局注册表中维护。
*   任务状态分为 `Running` 和 `Done`。

### 3.3 强制命令执行 (exec)
提供规避关键字冲突的逃生通道。
