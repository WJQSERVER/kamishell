# Kamishell 走向“生产可用”的路线图

为了使 Kamishell 成为一个真正可用且强大的跨平台 Shell，我们将以下事项作为当前开发的核心路径。

## ⚠️ 核心优先级：POSIX.1-2024 深度合规
**这是当前项目最优先的任务。** 我们的目标是让所有内置命令在行为、参数处理和错误响应上完全对齐最新的 POSIX 标准。

### 阶段 0: 内置命令合规化工程
- [ ] **全量审计**: 对照 POSIX.1-2024 规范文档，逐一审计以下命令：
  - `ls`, `rm`, `cp`, `mv`, `cat`, `mkdir`, `cd`, `pwd`, `grep`, `sed`, `env`, `export`, `touch`。
- [ ] **参数与行为重修**:
  - 完善所有必选和可选 Flag 的实现。
  - 统一错误消息格式，确保符合 POSIX 标准建议。
  - 严格校验退出码（Exit Codes）。
- [ ] **合规性测试套件**: 开发专门用于验证 POSIX 合规性的自动化测试。
- [ ] **标准对齐文档**: 在 `docs/` 下为每个命令建立合规性清单。

## 已完成阶段

### 1. 工程化与架构优化
- [x] **扁平化包结构**: 项目重构为扁平根包，包名统一为 `main`。
- [x] **跨平台基础**: 建立 Go 编译器多平台二进制构建能力。

### 2. 核心 Shell 基础设施
- [x] **管道 (`|`)**、**输出重定向 (`>`, `>>`)**。
- [x] **变量插值 (`$VAR`)**、**环境变量管理 (`export`)**。
- [x] **逻辑运算符 (`&&`, `||`)**、**异步执行 (`&`)**。

### 3. 跨平台内置工具集 (基础实现)
- [x] 已初步实现 `ls`, `rm`, `cp`, `mv`, `mkdir`, `cat` 等核心工具。
- [ ] **待增强**: 正在进行的阶段 0 将对这些工具进行 POSIX 深度对齐。

### 4. 编程能力增强
- [x] **流程控制**: `for` 循环、`func` 函数定义。
- [x] **并发支持**: `go` 关键字 Goroutine 支持。
- [x] **静态类型系统**: `var` 语句与 `:=` 类型推断。

### 5. 用户体验
- [x] **交互式 REPL**: Readline 集成（支持多后端）。
- [x] **自动补全**: 路径与命令补全。
- [x] **配置加载**: `.kamirc` 自动初始化。
