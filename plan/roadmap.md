# Kamishell 走向“生产可用”的路线图

为了使 Kamishell 成为一个真正可用且强大的跨平台 Shell，我们需要分阶段实现以下功能。

## 1. 核心 Shell 基础设施 (优先级：最高)
这些功能是现代 Shell 的基石，决定了其作为胶水语言的能力。

- [ ] **管道 (`|`)**: 支持将一个命令的 stdout 连接到另一个命令的 stdin。
- [ ] **输出重定向 (`>`, `>>`)**: 支持将结果保存到文件。
- [ ] **变量插值 (`$VAR`)**: 在命令参数或字符串中动态替换变量值。
- [ ] **环境变量管理**: 完整的 `export` 支持，并能将变量传递给子进程。
- [ ] **逻辑运算符**: 支持 `&&` 和 `||` 链式执行命令。

## 2. 跨平台内置工具集 (优先级：高)
为了保证在 Windows 和 Linux 下体验一致，我们需要用 Go 重新实现以下核心工具：

- [ ] `export` / `env`: 环境变量查看与设置。
- [ ] `cat`: 跨平台的文件读取输出。
- [ ] `mkdir` / `rm`: 目录创建与文件/目录删除。
- [ ] `touch`: 创建空文件或更新时间戳。
- [ ] `cp` / `mv`: 文件复制与移动。
- [ ] `exit`: 优雅退出 Shell（支持返回码）。
- [ ] `type` / `which`: 辨别内置命令与系统命令。

## 3. 编程能力增强 (优先级：中)
提升 Kamishell 编写复杂自动化脚本的能力。

- [ ] **循环结构 (`for`)**: 支持 `for i := range collection` 等 Go 风格迭代。
- [ ] **函数定义 (`func`)**: 支持定义可重用的代码块。
- [ ] **错误处理优化**: 确保 `if err != nil` 能够捕获所有内置和外部命令的异常状态。
- [ ] **算术运算**: 简单的 `+`, `-`, `*`, `/` 支持。

## 4. 用户体验 (优先级：低)
提升 REPL 的日常使用舒适度。

- [ ] **行编辑 (Readline)**: 支持历史记录、光标移动和快捷键。
- [ ] **自动补全**: 支持路径补全和命令名补全。
- [ ] **配置文件 (`.kamirc`)**: 启动时自动加载别名、环境变量和初始化脚本。
