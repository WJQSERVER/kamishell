# Kamishell 分词器 (Tokenizer) 实现说明

Kamishell 在不同的层面上使用了不同的分词规则，以平衡语法解析的严谨性与交互体验的便捷性。

## 1. 核心词法分析器 (Lexer)

位于 `lexer.go`，负责将输入的命令流转换为 AST (抽象语法树) 消费的 Token。

### 标识符 (IDENT) 规则
为了兼容 Shell 的使用习惯，Kamishell 的标识符判定比传统的编程语言更宽松：
- **允许字符**: 字母、数字、下划线 `_` 以及正斜杠 `/`。
- **用途**: 这使得裸露的路径（如 `/usr/bin/ls`）可以直接作为标识符解析，无需引号。
- **注意**: 连字符 `-` 和点 `.` 不属于标识符字符。例如 `apt-get` 会被解析为三个 token（`apt`、`-`、`get`），`apt.get` 会被解析为 `apt`、`.`、`get`。

### 自动分号插入 (ASI)
Kamishell 模拟了 Go 的行为，在以下 Token 之后遇到换行符会自动插入分号 (`;`)：
- 标识符、数字、字符串。
- 关键字: `true`, `false`, `nil`, `return`。
- 右括号/花括号: `)`, `}`。

### 字符串
- 仅支持双引号 `"`。
- 支持变量插值（在运行时通过 `os.Expand` 处理）。

---

## 2. Readline 交互层分词

位于 `readline`，处理 UI 交互时的单词操作。

### 单词边界 (Word Boundaries)
用于 `Ctrl+Left` (MoveWordLeft), `Ctrl+Right` (MoveWordRight) 以及 `Ctrl+Del` (DeleteWord)：
- **判定标准**: 使用 `unicode.IsSpace`。
- **逻辑**: 跨越连续的空格或连续的非空格字符。这意味着路径中的 `/` 或文件名中的 `.` 不会被视作边界，保证了路径操作的完整性。

---

## 3. 自动补全分词 (Completer)

位于 `completer.go`，决定 Tab 键补全时当前正在输入的“单词”。

### 分隔符规则
补全器使用以下字符作为单词的物理分隔符：
- 空格 (` `)、制表符 (`\t`)。
- 管道符 (`|`)、与符号 (`&`)。
- 重定向符 (`>`, `<`)。

**实现细节**:
```go
idx := strings.LastIndexAny(lineStr, " \t|&><")
```
补全器会从当前光标位置向左查找上述任意字符，截取之后的字符串作为待补全的原始前缀。

---

## 总结对比

| 功能层面 | 分隔符/边界依据 | 特点 |
| :--- | :--- | :--- |
| **Lexer (解析)** | 运算符、空白符、括号 | 支持路径字符作为标识符的一部分 |
| **Readline (移动/删除)** | `unicode.IsSpace` | 以空白为唯一边界，保护路径完整性 |
| **Completer (补全)** | ` \t\|&><` | 识别 Shell 操作符作为补全边界 |
